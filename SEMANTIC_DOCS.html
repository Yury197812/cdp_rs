Что такое Orchestrator LLM?

Система для массовой обработки задач в ChatGPT с управлением проектами, ветками и координацией.

Ключевая ценность: Управляемая обработка множества независимых рабочих веток без смешивания контекстов.

Как это работает?

Система использует модульную архитектуру на Rust:

orchestrator/ ├── core/ → ProjectManager, BranchManager, ChatManager, TaskManager ├── workers/ → Worker, WorkerPool ├── coordinator/ → DependencyMonitor, DataExchange, SyncManager └── dashboard/ → API (Axum), WebSocket, Stats

Зависимости: SQLite (rusqlite), Tokio, Axum

Query Fan-out паттерн

При запросе "Как использовать Orchestrator?" система генерирует:

  • → Как установить?
  • → Как настроить API?
  • → Как добавить проект?
  • → Как мониторить задачи?

Все ответы на одной странице — пользователь не уходит с сайта.

Session-based RAG

Модель удерживает контекст:

  • Сессия 1: "Установка" → ответ: Docker команды
  • Сессия 2: "API" → ответ: HTTP endpoints
  • Сессия 3: "Мониторинг" → ответ: WebSocket + Stats

Каждый ответ строится на предыдущем — связность контекста.
